# Filosofia — Un Grafo di Conoscenza

Quartz v5 site generated from the `VaultPhilosophy/` Obsidian vault (gitignored,
private source — the vault's text reaches this repo only via `content/`,
regenerated by `preprocess.mjs`).

## Build order

```sh
SPA=1 node preprocess.mjs
npx quartz plugin restore
NODE_OPTIONS=--max-old-space-size=14336 npx quartz build
node scripts/make-mobile-index.mjs
NODE_OPTIONS=--max-old-space-size=8192 node scripts/compress-search-index.mjs
```

- `preprocess.mjs` reads `../VaultPhilosophy` and `../data` (paths resolved relative
  to the script, never hardcoded), wipes and regenerates `content/` and
  `quartz/static/*.json`. **`SPA=1` is mandatory** — Cloudflare Pages caps a
  deployment at ~20,000 files, and the vault's ~9,320 atomized paragraphs would blow
  that cap as individual pages. In SPA mode every work is emitted as ONE reading page
  (`testi/<philosopher>/<work>.md`) with its atoms inline behind `atom-split` markers,
  routed client-side to `work#atom` fragments (`quartz/components/scripts/atomRouter.inline.ts`).
  Total output: ~115 reading pages + 187 aggregator notes (axes/positions/concepts/
  arguments/figures/forms/schools) + 5 generated pages ≈ 307 files.
- `npx quartz plugin restore` re-resolves the community plugins declared in
  `quartz.config.yaml` / `quartz.lock.json`.
- `npx quartz build` needs a large heap (~14GB) for the full corpus.
- `scripts/make-mobile-index.mjs` derives a light `contentIndexMobile.json` (title/
  tags/links + a short snippet) so mobile browsers don't OOM parsing the full index.
- `scripts/compress-search-index.mjs` shrinks the desktop `contentIndex.json` via
  TF-IDF (keeps a readable snippet + each page's top discriminating terms instead of
  its full text) — copied from `../English/quartz-eng-lit/scripts/` and unchanged
  (both scripts use only paths relative to the project root, so no adaptation was
  needed beyond the copy).

## Language-agnostic search

Every work is indexed under its taxonomy node **ids** (not just source-language
words) via `quartz/static/index.json`'s `axes[]`/`positions[]`/`concepts[]`/
`arguments[]`/`figures[]`/`forms[]`/`schools[]` arrays, and `quartz/static/
taxonomy.json` carries `label_it`/`label_en`/`aliases` for every id so the client can
render facets in either language. This is what makes a German or Latin passage
findable from an English query: the match happens on the canonical id, never on the
source-language prose.

## Data sources

- `VaultPhilosophy/Knowledge Graph/{Axes,Positions,Concepts,Arguments,Figures,Forms,Schools}/<id>.md`
  — the 187 aggregator notes (controlled-vocabulary nodes).
- `VaultPhilosophy/Philosophers/<Name>/_raw/<WORK>.md` — 115 works' publication
  metadata (title, edizione, traduttore, pd_year, ...).
- `VaultPhilosophy/Philosophers/<Name>/Atomized/<WORK>/<NNN>_<slug>.md` — 9,320
  atoms. **Gotcha:** on Windows, long `Atomized/<WORK>/` directory names are silently
  truncated (MAX_PATH mitigation baked into the atomizer). The directory name is
  therefore not a reliable work key — every atom's frontmatter carries the
  untruncated canonical key in `work:`, matching the `_raw` basename exactly.
  `preprocess.mjs` groups atoms by that field, never by directory name.
- `../data/taxonomy.json` — the controlled vocabulary (14 axes, 71 positions, 40
  concepts, 20 arguments, 17 figures, 14 forms, 11 schools).
- `../data/tags/<Philosopher>.json` — per-work tags (`axes`, `positions`, `concepts`,
  `arguments`, `figures`, `forms`, `schools`, `summary_it`, `summary_en`).
